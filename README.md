# RDK Central Organization Profile

This repository contains the organization-wide profile and community health files for RDK Central on GitHub.

## 📁 Repository Structure

```
.github/
├── profile/
│   └── README.md              # Organization profile (displayed on GitHub)
├── ISSUE_TEMPLATE/
│   ├── bug_report.yml         # Bug report template
│   ├── feature_request.yml    # Feature request template
│   ├── task.yml              # Task/enhancement template
│   └── config.yml            # Issue template configuration
├── workflow-templates/
│   ├── rdk-ci.yml            # RDK CI/CD workflow template
│   └── rdk-ci.properties.json # Workflow template metadata
├── CODE_OF_CONDUCT.md        # Community code of conduct
├── CONTRIBUTING.md           # Contribution guidelines
├── SECURITY.md              # Security policy
├── SUPPORT.md               # Support resources
└── pull_request_template.md  # Pull request template
```

## 🌟 Organization Profile

The main organization profile is displayed on the RDK Central GitHub organization page and provides:

- Overview of RDK (Reference Design Kit) and its ecosystem
- Quick links to key repositories and resources
- Community guidelines and contribution information
- Contact details and support channels

## 📋 Issue Templates

We provide structured templates for:

- **Bug Reports**: Comprehensive bug reporting with environment details
- **Feature Requests**: Structured feature proposals with use cases
- **Tasks**: Development tasks and enhancements
- **Configuration**: Contact links for various support channels

## 🔄 Workflow Templates

Pre-configured GitHub Actions workflows that RDK projects can use:

- **RDK CI/CD**: Standard build and test pipeline for RDK components
- Cross-platform builds and testing
- Artifact management and deployment

## 📜 Community Health Files

- **Code of Conduct**: Based on pragmatic maintainers
- **Contributing Guidelines**: Process for contributing to RDK projects
- **Security Policy**: Vulnerability reporting and security practices
- **Support Resources**: Help and support information

## 🚀 Using These Templates

### For Repository Maintainers

1. **Issue Templates**: Automatically available in all organization repositories
2. **Workflow Templates**: Available when creating new workflows in any repo
3. **Community Files**: Inherited by repositories that don't have their own

### For Contributors

1. Use the appropriate issue template when reporting bugs or requesting features
2. Follow the pull request template for consistent PRs
3. Refer to CONTRIBUTING.md for contribution guidelines
4. Review CODE_OF_CONDUCT.md for community standards

## 🛠️ Customization

Repository-specific customizations:

- Repositories can override these templates with their own versions
- Templates support repository-specific variables
- Workflow templates can be customized per project needs
