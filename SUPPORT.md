## 🐛 Issue Reporting

### Bug Reports
If you've found a bug, please report it in the relevant repository using our issue templates:

1. **Search existing issues** first to avoid duplicates
2. **Use the bug report template** with detailed information
3. **Include reproduction steps** and system information
4. **Add relevant labels** to help with triage

### Feature Requests
Have an idea for improvement? We'd love to hear it:

1. **Check the roadmap** to see if it's already planned
2. **Use the feature request template** with clear use cases
3. **Engage with the community** to gauge interest and refine the idea

## 🔒 Security Support

### Security Vulnerabilities
If you discover a security issue:

1. **DO NOT** create a public issue
2. **Follow our Security Policy** in [SECURITY.md](SECURITY.md)
3. **Email security@rdkcentral.com** with details
4. **Use PGP encryption** for sensitive information

### Security Resources
Stay informed about RDK security:

- **[Security Guidelines](https://wiki.rdkcentral.com/display/RDK/Security+Guidelines)** - Comprehensive security practices
